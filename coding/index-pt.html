Missão 1: O que são Algoritmos?

Um algoritmo é uma série de passos ordenados para resolver um problema ou completar uma tarefa. Usamo-los todos os dias sem sequer dar por isso!

Mestre Luciano diz: "Pensa num algoritmo como na lista de verificação que uso antes da descolagem! Cada passo deve ser executado na ordem certa para uma missão de sucesso."

Exemplos da vida real:

Missão 3: Como funcionam os computadores

Os computadores são máquinas extraordinárias que processam informações seguindo instruções precisas.

Mestre Luciano diz: "Pensa no computador como no nosso centro de controlo da missão: recebe informações, processa-as e produz resultados!"

Componentes principais de um computador:

CPU - O cérebro do computador

A CPU (Unidade Central de Processamento) é o cérebro do computador que executa as instruções dos programas.

Memória - A secretária de trabalho

A memória RAM é como uma secretária onde o computador guarda as informações que está a usar nesse momento.

1. O que faz a CPU num computador?
A. Mostra as imagens no ecrã.
B. Executa as instruções dos programas.
C. Armazena os ficheiros a longo prazo.
2. Que componente do computador é como uma secretária de trabalho?
A. O disco rígido.
B. A memória RAM.
C. A placa gráfica.

Missão 4: Programação em blocos

A programação em blocos é uma forma simples e visual de aprender a programar.

Mestre Luciano diz: "Os blocos são como os tijolos de construção: cada um tem uma função específica e juntos podem criar coisas incríveis!"

Tipos de blocos:

Blocos de movimento

Os blocos de movimento permitem mover os personagens no ecrã.

Blocos de controlo

Os blocos de controlo determinam quando e como são executadas as instruções.

1. Qual é a vantagem da programação em blocos?
A. É mais rápida que a programação textual.
B. É mais fácil de compreender para principiantes.
C. É mais poderosa que a programação textual.
2. O que fazem os blocos de controlo?
A. Mudam a cor do fundo.
B. Determinam quando executar as instruções.
C. Adicionam sons ao projeto.

Missão 5: Instruções condicionais

As instruções condicionais permitem ao computador tomar decisões com base em determinadas condições.

Mestre Luciano diz: "As condições são como bifurcações durante uma exploração: dependendo do que encontras, decides que caminho tomar!"

Exemplos de condições:

Se... então...

Se estiver a chover, então levo o guarda-chuva. Esta é uma condição simples que usamos todos os dias!

Se... então... senão...

Se tiver fome, então como, senão continuo a brincar. Esta condição tem dois resultados possíveis.

1. O que faz uma instrução condicional?
A. Executa sempre as mesmas ações.
B. Decide que ação executar com base numa condição.
C. Repete uma ação muitas vezes.
2. Qual destes é um exemplo de condição "se-então-senão"?
A. Se estiver a chover, levo o guarda-chuva.
B. Se estiver a chover, levo o guarda-chuva, senão ponho os óculos de sol.
C. Levo o guarda-chuva e ponho os óculos de sol.

Missão 6: Ciclos e repetições

Os ciclos permitem repetir uma ação várias vezes sem ter de a reescrever cada vez.

Mestre Luciano diz: "Os ciclos são como o motor de uma nave espacial: uma vez ligados, continuam a funcionar até os pararmos!"

Tipos de ciclos:

Ciclo "repetir por"

Repetir por 10 vezes: saltar. Este ciclo executará a ação de saltar exatamente 10 vezes.

Ciclo "repetir até"

Repetir até acabares os trabalhos de casa: estuda. Este ciclo continuará até a condição ser satisfeita.

1. Porque usamos ciclos na programação?
A. Porque são mais bonitos de ver.
B. Para evitar repetir o mesmo código muitas vezes.
C. Porque os computadores os preferem.
2. Que ciclo se repete um número específico de vezes?
A. Repetir por.
B. Repetir até.
C. Repetir sempre.

A História dos Algoritmos

A palavra "algoritmo" deriva do nome do matemático persa Al-Khwarizmi, que viveu no século IX. Os seus livros introduziram os números indianos (os que usamos hoje) e os métodos para resolver equações.

Ao longo dos séculos, muitos matemáticos contribuíram para o desenvolvimento dos algoritmos, mas foi com o advento dos computadores que os algoritmos se tornaram fundamentais para a nossa vida quotidiana.

Curiosidade: O primeiro algoritmo informático foi escrito por Ada Lovelace em 1843 para a Máquina Analítica de Charles Babbage, muito antes de existirem computadores modernos!

Como funciona um compilador

Um compilador é um programa especial que traduz o código escrito numa linguagem de programação (como Python ou Java) em linguagem de máquina, que o computador pode compreender e executar diretamente.

O processo de compilação ocorre em várias fases:

  1. Análise lexical: o código é decomposto em "tokens" (palavras-chave, identificadores, etc.).
  2. Análise sintática: verifica se os tokens estão organizados corretamente de acordo com as regras da linguagem.
  3. Análise semântica: verifica se o código tem um significado lógico.
  4. Geração de código: produz o código em linguagem de máquina.

Mestre Luciano diz: "Pensa no compilador como num tradutor universal que traduz as tuas ideias para uma linguagem que o computador consegue entender!"

Mulheres importantes na tecnologia

Para além de Ada Lovelace e Grace Hopper, muitas outras mulheres deram contributos fundamentais para a informática e tecnologia:

  • Hedy Lamarr: Atriz e inventora, desenvolveu uma tecnologia que está na base do Wi-Fi e Bluetooth.
  • Radia Perlman: Conhecida como "a mãe da Internet", inventou o protocolo spanning-tree.
  • Anita Borg: Fundou o Institute for Women and Technology e a Grace Hopper Celebration.
  • Margaret Hamilton: Dirigiu o desenvolvimento do software para o programa Apollo da NASA.

Sabias? As mulheres foram fundamentais no desenvolvimento da informática desde o seu início, e continuam a fazer descobertas revolucionárias em todas as áreas da tecnologia!
